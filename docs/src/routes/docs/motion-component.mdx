# Motion Component

The `motion` proxy exposes a `motion.*` component for every HTML and SVG element.
Motion components accept all native props plus motion-specific props.

## Basic usage

```tsx
import { motion } from "motion-solid";

<motion.div
  class="circle blue"
  initial={{ opacity: 0, scale: 0 }}
  animate={{ opacity: 1, scale: 1 }}
>
  I'm a motion.div
</motion.div>;
```

## Core props

### `initial`

Initial state of the component. Can be:

- An animation target
- A variant label (see [Variants](/docs/variants))
- `false` to skip the initial animation

### `animate`

Target to animate to on mount and update. Can be:

- An animation target
- A variant label or array of labels

### `exit`

Target to animate to on exit. Can be an animation target or a variant label.
This only runs inside [AnimatePresence](/docs/animate-presence).

### `transition`

Default transition to use when a target does not specify one.
`transitionEnd` values are applied after animations complete.

### `variants`

Map of variant names to animation targets. See [Variants](/docs/variants).

### `custom`

Custom data passed to variant functions. This can also be supplied by
[AnimatePresence](/docs/animate-presence).

### `inherit`

Set to `false` to prevent inheriting variants from ancestors.

### `style`

Regular Solid style object merged with Motion's rendered styles.

### `transformTemplate`

Function that lets you control the final `transform` string.

### `onUpdate`

Callback invoked with the latest resolved values on every render.

### `onAnimationStart` and `onAnimationComplete`

Callbacks fired when the `animate` target starts or finishes.

## Gestures and drag

Motion supports `whileHover`, `whileTap`, `whileFocus`, `whileInView`, and
`whileDrag` along with their corresponding event callbacks. See
[Gestures](/docs/gestures) and [Drag](/docs/drag).

## Layout

Set `layout` or `layoutId` to enable layout animations, and use
`layoutDependencies` for Solid-specific reactivity. See [Layout](/docs/layout).

## Server-side rendering

On the server, motion components render with the styles from the `initial` prop
if provided, or the `animate` prop if `initial` is `undefined` or `false`.

```tsx
// Server will output `translateX(100px)`
<motion.div initial={false} animate={{ x: 100 }} />
```

```tsx
// Server will output `opacity: 0; transform: scale(0);`
<motion.div
  initial={{ opacity: 0, scale: 0 }}
  animate={{ opacity: 1, scale: 1 }}
/>
```

## Planned

- Custom components (`motion(Component)`) are not supported yet.
- MotionValue support in the `style` prop is planned.
