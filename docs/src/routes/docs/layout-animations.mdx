# Layout Animations

`motion-solid` exposes the same core layout surface as Motion: `layout`, `layoutId`, `layoutDependency`, `layoutScroll`, `layoutRoot`, `layoutCrossfade`, `LayoutGroup`, `useInstantLayoutTransition`, and `useResetProjection`.

## Automatic Layout

Use `layout` when the same element should animate between size and position changes:

```tsx
<motion.div layout />
<motion.div layout="position" />
<motion.div layout="size" />
<motion.div layout="preserve-aspect" />
```

`layout="position"` is the escape hatch for text and other content that should move without being visually scaled.

## Shared Layout

Use `layoutId` when one visual element should hand off to another:

```tsx
<LayoutGroup id="cards">
  <motion.button layoutId="card-shell" />
  <Show when={open()}>
    <motion.div layoutId="card-shell" />
  </Show>
</LayoutGroup>
```

If you need a whole card handoff, put the shared `layoutId` on the card shell itself, not only on one inner row.

## External State And `layoutDependency`

Solid only re-runs a motion host when that host actually reads a signal. If layout needs to react to related state that lives outside the node, pass it through `layoutDependency` explicitly:

```tsx
<motion.button layout layoutId="card-shell" layoutDependency={selectedId()} />
```

This is especially important for shared-layout patterns where a selection signal mounts a second element elsewhere in the tree.

## Distortion And Scale Correction

Motion’s layout projection works by animating transforms. That means three rules matter:

1. If immediate children look stretched, add `layout` or `layout="position"` to those children too.
2. `borderRadius` and `boxShadow` correction only works when Motion can see those values on the motion component itself.
3. For that reason, prefer inline `style`, `initial`, `animate`, or `exit` values for corrected surfaces instead of only utility classes.

```tsx
<motion.div
  layout
  style={{
    "border-radius": "24px",
    "box-shadow": "0 18px 40px rgba(15, 23, 42, 0.16)",
  }}
/>
```

## Scroll Roots And Portals

Use the Motion projection options directly when layout lives in scroll containers or custom roots:

```tsx
<motion.div layout layoutScroll />
<motion.div layout layoutRoot />
<motion.div layoutId="modal" layoutCrossfade={false} />
```

## Reordering

Sibling reordering in Solid control flow is supported. A typical list reorder looks like this:

```tsx
<motion.div layout>
  <For each={sortedItems()}>
    {(item) => <motion.div layout>{item.title}</motion.div>}
  </For>
</motion.div>
```
